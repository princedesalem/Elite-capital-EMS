## 🎮 Gestion Post-Installation

### Contrôle les Services

```powershell
# Toutes les commandes depuis : C:\EMS\extranet

# Démarrer l'application
.\manage-ems.ps1 start

# Arrêter l'application
.\manage-ems.ps1 stop

# Voir les containers en cours
.\manage-ems.ps1 status

# Logs en temps réel (appuyez Ctrl+C pour quitter)
.\manage-ems.ps1 logs

# Redémarrer l'application
.\manage-ems.ps1 restart
```

### Backups Automatiques

```
✅ Chaque nuit à minuit : backup quotidien auto
✅ Stocké dans : C:\EMS\backups\ems-db-YYYY-MM-DD-HHMMSS.sql
✅ Nettoyage auto : supprime backups > 30 jours

# Voir les backups
dir C:\EMS\backups\

# Restaurer un backup (si besoin)
docker compose exec db mysql -u extranet -pextranet EMS_DB < \
  C:\EMS\backups\ems-db-2026-05-15-000000.sql
```

### Configuration CI/CD (Auto)

```
✅ GitHub Actions runner installé automatiquement
✅ Deploy job programmé dans .github/workflows/ci.yml

Workflow :
1. Développeur git push origin main
2. GitHub Actions lance tests (pytest + vitest)
3. Si tests OK → redéploiement automatique du serveur
4. Si tests FAIL → pas de déploiement (sécurité)

Logs : https://github.com/princedusalem/Elite-capital-EMS/actions
```

## 🐛 Si Ça Ne Fonctionne Pas...

### Application ne démarre pas
```powershell
# 1. Voir l'état
.\manage-ems.ps1 status

# 2. Voir les erreurs
.\manage-ems.ps1 logs | head -50

# 3. Redémarrer proprement
.\manage-ems.ps1 stop
.\manage-ems.ps1 start
```

### Port déjà utilisé (5173 ou 8000)
```powershell
# Redémarrer Docker
.\manage-ems.ps1 stop
Start-Sleep -Seconds 5
.\manage-ems.ps1 start
```

### Base de données ne connecte pas
```powershell
# Tester connexion
docker compose exec db mysql -u extranet -pextranet -e "SELECT VERSION();"

# Voir les logs DB
docker compose logs db | head -50
```

### GitHub runner ne se lance pas
```
Check : https://github.com/settings/actions/runners
Logs  : C:\actions-runner\_diag\
Solution : Relancer l'installateur ou contact support
```
